# LoROMアドレス補正メモ v1

## 結論

新桃ROMの今回解析対象は、LoROM換算で扱うのが正しい。

根拠は、既知の `C0:BD98` decoder が file PC `0x003D98` に対応すること。
HiROM換算ではROMサイズ外/不整合になる。

## 主要ラベル修正

| 旧表記 | 正しい表記 | file PC |
|---|---|---:|
| `$83:9850` | `$87:9850` | `0x039850` |
| `$83:F09A` | `$87:F09A` | `0x03F09A` |
| `$83:F0DB` | `$87:F0DB` | `0x03F0DB` |
| `$84:89C7` | `$89:89C7` | `0x0489C7` |
| `$84:9BEE` | `$89:9BEE` | `0x049BEE` |
| `$84:8078` | `$89:8078` | `0x048078` |
| `$84:1A10` | `$88:9A10` | `0x041A10` |

## 注意

過去の成果物の内容自体、たとえば `0x39850` や `0x3F09A` というfile PC上の関係は有効。
ただしCPU bank表記はLoROMに補正する必要がある。
