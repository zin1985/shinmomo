## 0. 統合要約

- 対象ROMサイズは **2,097,152 bytes (0x200000)**。
- β版と通常版で重要候補の大半は一致し、**`0x41AC0` 周辺だけ規則的差分**が目立つ。
- **`0x442BF` 付近は道具表本命**、**`0x449D0` 付近は装備表本命**。
- **`0x41A10..` は 8バイト固定長の分岐表本命**で、`0x31DDF` や `0x320A4` などのスクリプト帯へ飛ぶ。
- **`0x31C68 / 0x31DF8 / 0x320C8` は店/施設スクリプト帯候補**。ただし `0x31C68` 直結は未発見。
- **`0x6B340..0x6B629` は bank86 共通UI状態機械**。`$1FB0..$1FB9` は UIアニメ用10バイト作業領域。
- **`0x26A60..0x26C3F` は店在庫本体説が後退**し、UI/描画/表示定義寄りに修正。
- **`0x4F` は会話分岐より UI/施設の field抽出命令** として理解するのが安全。
- **`7E:718B..` は表示コード staging / operand stack 寄り**で、tilemap / DMA 直結は未確定。
- 武器特殊能力は **店/施設本線とは独立した小VM系** として別枠管理が妥当。

## 1. 重要テーマ別の統合メモ

### 1-1. 表データ
- 道具表本命: `0x442BF` 付近、11バイト固定長仮説。
- 装備表本命: `0x449D0` 付近、16バイト固定長仮説。
- 装備系列の手応えが高い範囲:
  - `0x449D0..0x44A80` 桃太郎武器帯
  - `0x44B20` 周辺 胴装備帯
  - `0x44E50..0x44EC0` 杖帯
  - `0x44FA0..0x45020` バラ帯

### 1-2. 分岐・スクリプト
- `0x41A10..` は 8バイト固定長の分岐表本命。
- レコード形式仮説: `[key][c1][c2][c3][c4][c5][target_lo][target_hi]`。
- `target` は `0x30000 + target` とみると自然で、`0x31DDF` / `0x320A4` へつながる。
- `0x31DF8` は `0x31DDF` に入った後の同ストリーム途中とみるのが自然。
- `0x31C68` への直結は未発見。

### 1-3. UI / WRAM
- `0x6B340..0x6B629` は共通UI状態機械。
- `0x6B68A` は 10バイト構造体を `$1FB0..$1FB9` にコピー。
- `$1F9D/$1F9E` は店/施設フェーズ値系、`$1FAA` はサブ機能セット番号が有力。
- `0x69FB6 = FF 03 00 00 01 02 01 02` は段階値変換表候補。

### 1-4. 表示VM / field抽出
- opcode `0x4F` → handler `89:9A44`。
- selector 1/2/3 は 商品系 / 装備系 / キャラ系 が有力。
- `89:845E` の結果はまず `($9C)` が指す作業先RAMへ書かれる。
- `7E:718B..` は表示コード staging / operand stack とみるのが安全。

### 1-5. 未解決の本丸
- 所持金RAM本体
- `0x449D0` / `0x442BF` を実際に読むコード
- 店価格比較 / 購入可否 / 減算本線
- `0x31C68` 接続
- `0x41A10..` を実際に走査比較する本体コード
- NPC会話起点 → 施設判定 → 店本線
