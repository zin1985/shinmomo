# NPC大量表示軽減パッチ feasibility note（vol016）

## 結論

試作パッチは作れる。正式パッチは追加検証が必要。

## 試作パッチ候補

### 候補A: object生成入口でskip

```text
C0:AF33 object生成/insert入口で、
$0AE5 active count が閾値以上、かつ sort key が低いobjectのみ生成skip
```

メリット:
- 生成・sort・OAM展開の前で止められるため軽量化効果が大きい。

リスク:
- イベント用objectや同行者を消すと進行不能化する可能性。

### 候補B: OAM build時だけskip

```text
C0:B03D active chain walk / C0:B100 OAM piece展開時に低優先度objectだけ描画skip
```

メリット:
- 内部object状態は残るためイベント破壊リスクが低い。

リスク:
- object処理/ソートコストは残る。

## 初回パッチ条件案

```text
if active_count >= 0x30 and sort_key < low_priority_threshold:
    skip_draw_or_spawn
```

最初は描画skip側が安全。次に生成skipへ進む。

## 検証項目

- 始まりの村
- NPC多めの町
- 屋内
- イベント中
- 同行者あり
- 戦闘前後

ログ比較:

```text
$0A61 active chain
$0AA3 sort key
$0AE5 active count
$0EE9 OAM mirror
visible_objects.csv
```

## 未確定で危ない点

- C0:AF33 callerごとのhandle保存先
- C0:B100とB294 tableの完全接続
- $0B27 bit定義
- $0A1C OAM budget単位
- C1:90ED周辺
- $157D, $15E9,X, $6A の意味
