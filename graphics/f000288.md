## 2. 重要な結論

### 2.1 マップチップ復元

現状は、**VRAM snapshot からフィールドの仮マップチップを抽出・可視化できる段階**。

確定寄りの仮設定:

```text
tilemap base : VRAM 0x0000
char base    : VRAM 0x8000
bpp          : 4bpp
unit         : 2x2 tile = 16x16 px
```

今回の成果物:

```text
graphics/field/
graphics/mapchip_samples/
```

特に `shinmomo_field_mapchip_sample_top16_large.png` と `shinmomo_field_mapchip_sample_top64.png` は、16x16単位の仮マップチップ確認用。

### 2.2 カラー復元

正確なカラー再現には `CGRAM` が必要。  
白黒・疑似濃淡の形状確認は済んでいるが、正確な色付き復元は以下が揃ってから進める。

```text
vram.bin
cgram.bin
tilemap_pages.csv
state_frame.txt / PPU register dump
```

今後のLuaでは `$2105`, `$2107..$210C`, `$212C`, `$212D` を必ずstateに残す。

### 2.3 町・キャラクターの再現

町マップチップはフィールドと同じ手順で復元可能。  
ただし町専用dumpが必要。

キャラクター/NPCはBGではなくOAM/sprite系として分ける。

```text
BG / mapchip layer:
  VRAM tilemap + char + CGRAM + PPU state

Sprite / NPC layer:
  OAM mirror + visible object chain + sprite CHR + CGRAM
```

### 2.4 NPC大量表示軽減パッチ

Goal 13 が高進捗なので、**試作パッチは作れる段階**。  
ただし、正式配布レベルの安全パッチには追加検証が必要。

安全な初手:

```text
C0:AF33 object生成入口で、
active object数が多いときだけ、低priority/sort key objectを生成skipする実験IPS
```

候補観測先:

```text
$0A61 active linked list
$0A1F prev link
$0AA3 sort key / priority / depth key
$0AE5 active count
$0EE9 OAM mirror
C0:AF33 object生成/insert入口
C0:B03D active chain walk + OAM build
C0:B100 1 object -> OAM pieces
C0:B0C7 OAM DMA転送
```
