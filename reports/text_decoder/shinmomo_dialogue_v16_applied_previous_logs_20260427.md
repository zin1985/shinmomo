# v16 静的辞書補助を先ほどのログ片に適用した結果

※ここでの適用は、会話中に貼られた `linebuf / raw_jp_tokens / window_tokens / observed_tokens` 断片に対する再解釈です。エミュレータ上でv16を再実行した結果ではありません。

## Case A: たき火の村人

### 画面上の正解参照

```text
「桃太郎さん！　たき火に
　あたって　いきなよ！
　あったかいよー！」
```

### 入力ログ片

```text
raw_jp_tokens=!!!ん! き?ににににあたっ いきよ!!!!あったいよー」 / window_tokens=$12B5=1801:太
```

**v16 direct_general**

```text
！ん！ き?ににににあたっ いきよ！あったいよー」
```

**v16 observed tokens used**

```text
1801:太（単独）
```

**v16 static/compound candidates**

```text
太（桃太郎は未確定。1800/1802または02A0が必要）
```

**v16 decoded view**

```text
！ん！ き?ににににあたっ いきよ！あったいよー」 + 候補: 太
```

**評価**: 固定置換なしだと桃太郎・火はまだ不足。rawのかな骨格は取れるが、名前・漢字補完は観測token不足。

**不足/次に必要**: 02A0=桃太郎 または 1800+1801+1802、188F=火、欠落かな「た」「て」「な」「か」

## Case B: おむすびころりんの穴・旅の人

### 画面上の正解参照

```text
「桃太郎さん！　えらいこっちゃ！
　お宮って　いきなし穴が　鬼に
　おそわれたと　旅の人が　言うてただ！」
```

### 入力ログ片

```text
linebuf=さん えらこちゃ!!!おむすびこりのがぢにににおそわた 旅の人がてただ
```

**v16 direct_general**

```text
さん えらこちゃ！おむすびこりのがぢにににおそわた 旅の人がてただ
```

**v16 observed tokens used**

```text
029B:旅の人（暫定）, 1A1A:言（過去windowで観測）
```

**v16 static/compound candidates**

```text
旅の人 / 言 / 桃太郎?（桃系tokenが同時観測されれば） / 鬼・穴はtoken観測が必要
```

**v16 decoded view**

```text
さん えらこちゃ！おむすびこりのがぢにににおそわた 旅の人がてただ + 候補: 旅の人 / 言
```

**評価**: 旅の人・言は補える。桃太郎、穴、鬼は静的辞書には入ったが、このログ片では観測不足。

**不足/次に必要**: 02A0/1800-1802、18DB=鬼、19EB=穴、欠落かな「い」「っ」「ろ」「ん」「れ」「と」

## Case D: 北西のおむすびころりんの穴

```text
「北西の　おむすびころりんの穴は

```

**v16 direct_general**


**v16 observed tokens used**

```text

```

**v16 static/compound candidates**


# 総評

v16を先ほどのログ片に適用すると、固定置換ではなく、観測された漢字・辞書tokenから以下の候補が出せる。

- `旅の人`
- `言`
- `行く`
- `入れ / 入れません`
- `北`
- `桃太郎?`
- `銀 / 銀次? / 銀次のそうび?`

一方で、全文復元にはまだ不足が残る。特に以下は `$12B2/$12C4` 表示作業窓だけでは落ちやすい。

- `桃太郎` の完全形
- `銀次` の `次`
- `刀`
- `穴`
- `鬼`
- `体`
- `無理`
- 欠落かな（ろ、ん、て、な、か、まい等）

次の本命は `source_state=$1264..$1286` から原文reader / source token pointerを追うこと。
