# C0:BD98 mode02 decoder static analysis

## 結論

`$12AA=02` は、圧縮/特殊sourceというより **bitstream + binary tree table decoder** だった。

これが原因で、`C8:A7DD` のrawを1byte文字列として読むと壊れていた。

## なぜ「桃太郎さん」が断片化したか

1. `$12AA=02` では、本文がbitstreamから1symbolずつ復元される。
2. `桃太郎` は `02 A0` または `18 00 18 01 18 02` のような複合tokenとして復元される。
3. `$12B2/$12C4` は表示stageであり、復元済みtokenの一部だけが見えることがある。
4. pollingでは連続tokenを取りこぼしやすいため、`さん`だけ、`郎`だけ、`太`だけになる。

## 実測的な確認

Pythonで `C8:A7DD` からBD98展開を試したところ、以下が得られた。

```text
「桃太郎! 銀次の そうびを
 ととのえたか?
 銀次は 刀も そうびできる!」
「銀次の そうびは 着流しだ!
 はちまきと わらじは
 桃太郎と いっしゅだがな!」
```

`いっしゅ` は文字コード表側の未解決/誤割当の可能性が残る。  
ただし mode02 の大枠はBD98展開で正しい。
