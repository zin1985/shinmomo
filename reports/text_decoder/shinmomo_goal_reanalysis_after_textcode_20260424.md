## 1. 今回の結論

文字コード解析の成果を踏まえると、次に伸ばしやすいゴールは以下の順。

1. **9. 会話・店・イベントスクリプトの仕様書を作る**
   - 文字列帯の用途分類がかなり進み、script本体未解読でも「どの帯が何の処理か」を仕様化しやすい。
2. **5. 店・施設UIのレコード構造を確定する**
   - `0x0823CD..0x0835FF` の店・施設・宿屋・医者・預かり系メッセージが読めてきたため、既存の ledger / choice / transaction 解析と接続できる。
3. **11. アイテム・装備・キャラ・会話の外部データ化**
   - 542件級の文字列総覧、クイズ88問、短反応、後半会話帯などが既に正本化されており、CSV/MD化の足場が強い。
4. **12. 新桃全体の構造を人間可読で再構成する**
   - ROM配置がシナリオ順ではなく、戦闘会話・商店施設・ミニゲーム・マップイベント・クイズなどの機能クラスタ単位で整理できる。
5. **8. 条件分岐ディスパッチ系の確定**
   - `41A10 reader` 本体はまだ未発見だが、target側テキスト帯の意味が増えたことで、表レコードの target 意味づけがしやすくなった。

NPC大量表示軽減は前回進めた `C0:B100..B7A6` 系が本線で、文字コード解析からの直接伸びは小さい。ただし「キャラ表示資材」「speaker-label」「短反応の話者対応」が進んだので、NPC/キャラ資材外部化側から間接的には支援できる。

## 2. 文字コード解析で確定した前提

重要な上書き確定:

- `50 = 空白`
- `5B = ?`
- `5C = !`
- `5D = 、`
- `5E = 。`
- `5F = …`
- `A8 = の`
- `A9 = は`
- `BE = で`
- `18xx / 19xx / 1Axx / 1Bxx` は 2byte 漢字コード
- `02xx` は表示文字表の外側にある圧縮辞書 / 参照句候補

これにより、過去の以下の見立ては修正する。

- `5B = packet終端` 説は撤回。実体は疑問符。
- `50 = delimiter / 制御` 説は撤回。実体は空白。
- `A8/A9/BE` は制御ではなく普通の助詞/かな。
- 静的デコードで崩れる場所は、文字コード間違いよりも `02xx` 辞書・動的差し込み・別圧縮・script制御混入を疑う。

## 3. 今回の追加静的解析

### 3-1. `0x083100..0x0831FF` 医者 / 治療料金帯

見える芯:

- `医者`
- `なにか ごよう`
- `手おくれ`
- `なおす`
- `両で かかる`
- `チチンプイ`

解釈:

- 医者受付、状態異常治療、治療料金提示、治療実行、治療不可のメッセージ帯。
- `83:D906` などの購入/支払い core とは別に、治療料金も `$1E/$1F` amount helper へ流れる可能性が高い。
- 店・施設UIの「会計/支払い/確認」共通メッセージとして、Goal 5 に寄与。

### 3-2. `0x083200..0x0833FF` 所持金不足 + 宿屋帯

見える芯:

- `お金が 足りないことに 気づいた!`
- `申し訳ございません!`
- `いっぱいなんです`
- `ひとばん`
- `両`
- `ごゆるりと`
- `おはようございます`
- `野宿`

解釈:

- 前半は支払い不能・所持金不足・宿泊拒否。
- 後半は宿屋の料金提示、宿泊実行、翌朝挨拶、場合によっては野宿/満室/人気度条件のような分岐文。
- 施設UIの共通フローとしては、`提示 -> 確認 -> 所持金判定 -> 実行/拒否 -> 結果文` のテンプレートに載る可能性が高い。

### 3-3. `0x083400..0x0835FF` 人気度 / 解禁 / 探索反応

見える芯:

- `人気度が 80をこえた`
- `宿屋に とまれるようになった`
- `どうしちゃったの?`
- `にげてばかりじゃ だめ`
- `ちかごろ 鬼たいじを わすれて わるさ してませんか?`
- `桃太郎は かけじくを ながめた!`
- `しかし なにも わからなかった`
- `世の中 銭や!`

解釈:

- 人気度しきい値による施設解禁、低評価/逃走/悪行気味への反応、掛け軸調査などのシステム文が混在。
- 施設フローだけでなく、イベント探索・人気度システムのメッセージ帯として切るのがよい。

### 3-4. `0x082D00..0x0830FF` 関西弁商人 / 十一屋 / 預かり / 金庫

見える芯:

- `おこしやす`
- `えらいこと`
- `ごっつう 役に立ちまっせ`
- `十一屋`
- `いくら あずけ`
- `引き出し`
- `もちものを えらんで`
- `ナニモハイッテマセンヨ`
- `きんこ`
- `十一屋グループのしくみを 説明しまひょ`
- `650両まで`
- `すんません`

解釈:

- 関西弁商人、預かり/引き出し、金庫、十一屋説明が連結した施設帯。
- `ledger` 系とは別に、預かり金/金庫容量/引き出し制限のサブシステムがある可能性。
- Goal 5 の「店・施設UI」は、店/装備だけでなく、医者・宿屋・金庫/十一屋まで含む施設メッセージ集積帯として拡張すべき。

### 3-5. `0x09BF20..0x09C8FF` 怨みの洞窟 入力・動的差し込み帯

新しく見えた芯:

- `いちばん きらいな ...`
- `続いて 2番目に ...`
- `最...`

解釈:

- `0x09BE1D..0x09BF1F` で確定していた「嫌いな人の名前を3人入力する洞窟」の続きとして自然。
- `0x09BF28..0x09BF5A` は、1人目・2人目・3人目の名前入力プロンプト本体に近い。
- その後の `0x09C010..0x09C8FF` は静的デコードでは崩れが強い。名前入力結果の動的差し込み、入力バッファ参照、別辞書、選択肢制御が混ざるとみる。
- Goal 9 と Goal 11 に加え、動的差し込みのscript仕様化として Goal 1 / Goal 8 にも少し接続する。
