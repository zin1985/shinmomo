# 武器特殊能力サブシステム ワーキング仮説（2026-04-30）

## 位置づけ

このファイルは、vol014スレッド内で非イベント系の伸びしろとして選んだ Goal 7「武器特殊能力サブシステム」の作業メモである。

**注意:** ここに書く内容は、現時点では確定解析ではなく、次スレッド・別スレッドで検証するための仮説セット。

## 既存資料から確実に言えること

- 装備表は `0x449D0` 付近、またはschema基準では `0x449C3` 寄りに存在する。
- 装備表は16バイト固定長として実用ダンプ済み。
- 実用フィールドとして、`+00..02` は種別/特殊フラグ、`+0E` は装備スロット、`+0F` は能力上昇値と整理済み。
- 2026-03-18公式引継ぎでは、武器特殊能力サブシステムは 18% で、本格解析未着手扱い。

## 今回スレッドで出た仮説

### 仮説1: 装備表に特殊カテゴリIDがある

候補:

```text
装備表 +00..02 = 種別/特殊フラグ群
```

この中の1バイトまたはbitfieldが、戦闘中の武器特殊カテゴリIDとして参照される可能性。

### 仮説2: 戦闘処理内にCMP連鎖ディスパッチがある

想定形:

```asm
LDA weapon_special_id
BEQ normal_attack
CMP #$01
BEQ effect_01
CMP #$02
BEQ effect_02
CMP #$03
BEQ effect_03
```

現時点では実アドレス未確定。

### 仮説3: 武器特殊は即時処理と遅延処理に分かれる

想定:

```text
即時処理: ダメージ補正、追加ダメージ、倍率補正
遅延処理: 状態異常、追加攻撃、特殊演出、後段処理フラグ
```

### 仮説4: `7E:30xx` は戦闘効果の一時バッファかもしれない

今回スレッドでは `7E:3030` などを状態フラグとして扱う仮説が出た。ただし既存引継ぎ上では `7E:3004..303C` はD系opcodeの宛先例として挙がっており、UI/画面中間構造体の可能性もある。

そのため、戦闘WRAMと断定せず、以下を確認すること。

- 戦闘中だけ書かれるか
- UI/施設画面でも書かれるか
- 読み出し側が戦闘処理か
- `STA $3030` / `BIT $3030` / `ORA #imm` などの実アドレス

### 仮説5: `7E:20xx` への転送は未確定

今回スレッドでは `7E:30xx -> 7E:20xx` の状態定着モデルを出したが、これは未検証。

次回確認すべきこと:

```text
1. 7E:30xx を読む処理の実アドレス
2. その処理が 7E:20xx に書くか
3. 7E:20xx がキャラ/敵の恒常状態テーブルか
4. 状態異常表示やターン終了処理と接続するか
```

## 次の具体的な探索手順

1. GitHub最新の `data/weapon_special/` を読む。
2. 装備表CSVから、武器スロット `+0E=00` の行だけ抽出。
3. `+00..02` の値が非ゼロ・特徴的な武器をリスト化。
4. 逆アセンブルから、装備表baseまたは `0x449C3/0x449D0` に相当する参照を探す。
5. 戦闘処理内で装備IDを読む箇所を特定。
6. 非ゼロ特殊フラグをAレジスタに乗せる箇所を探す。
7. その直後のCMP/BIT/JSR/STA先を確定。
8. Luaで実行時トレースし、武器変更で分岐が変わるか確認。
